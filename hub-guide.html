딱 3가지 개념만

옵시디언(Obsidian) = 메모를 폴더에 파일로 저장하는 무료 글쓰기 앱. 아래 3개만 알면 충분해요.

🗄️
볼트 (Vault)
내 메모가 전부 담기는 폴더 하나. "여기가 내 서재" 라고 정해준 폴더예요. 야간비행님은 C:\Deeper-era.
📄
노트 (.md 파일)
글 한 편이 파일 하나. 확장자가 .md(마크다운 — 그냥 글자 파일이에요). 워드처럼 무겁지 않아 가볍고 안 깨져요.
🔗
링크 [[ ]]
글 속에서 [[다른노트]] 처럼 대괄호 두 개로 노트끼리 연결. 이 연결들이 모이면 관계도(그래프)가 됩니다.
참고 — 옵시디언에는 원래 '그래프 뷰'라는 기능이 있어요. 노트들이 별처럼 뜨고 [[링크]]가 선으로 이어진 관계 그림을 보여줍니다. 허브는 그 관계를 손님도 보기 좋게 한 번 더 다듬어 무대에 올린 것이라고 이해하면 됩니다.
가장 많이 묻는 것

클로드와 옵시디언, 어떻게 연결하나요?

결론부터: 별도의 '연결 장치'는 없어도 됩니다. 볼트는 그냥 폴더라서, 클로드가 그 폴더를 직접 읽어요.

핵심: 옵시디언 볼트는 사실 글자 파일이 든 폴더 하나예요. 그래서 클로드 코드는 그 폴더를 곧바로 읽고 씁니다. "이 폴더 읽어줘"라고 말하는 것, 그게 바로 연결입니다.
연결 방식어떻게 하나허브 만들기엔
① 폴더 직접 읽기
(클로드 코드)		볼트 폴더 경로만 알려주면 끝. 옵시디언 앱을 켜둘 필요도 없음✅ 권장 · 지금 이 방식
② 옵시디언 MCP/플러그인켜져 있는 옵시디언 앱과 실시간으로 대화(앱의 검색·태그 기능까지 사용)선택 · 필수 아님

후배는 이 한마디만 하면 됩니다

"내 옵시디언 볼트는 C:\내서재 야. 이 폴더를 읽어줘."

이렇게 폴더 위치를 알려주는 순간 클로드가 그 안의 노트를 전부 훑어볼 수 있어요. 뒤에 나오는 프롬프트 ①(볼트 진단)이 작동하는 이유가 바로 이것입니다.

더 쉬운 방법 — 데스크탑 클로드 코드를 볼트 폴더에서 바로 열면, 경로를 말할 필요도 없이 그 폴더가 기본 작업 폴더가 됩니다. (야간비행님은 이미 이렇게 쓰고 계세요.)

직접 연결하기 — 실제 클릭 4단계

처음 세팅하는 후배가 손에 잡히게 따라올 순서예요. 준비물: 옵시디언(무료) + 클로드 데스크탑 앱(Pro 이상) + 윈도우 10 / 맥 13 이상.

1

옵시디언 설치 → 볼트 만들기

obsidian.md 에서 내려받아 설치해요.

앱을 열고 'Create new vault'로 볼트 이름을 짓습니다(예: 내서재). 저장 위치는 '문서(Documents)' 폴더를 고르면 찾기 쉬워요. 이 폴더가 앞으로 '창고'가 됩니다.

2

클로드 데스크탑 앱 설치 → Code 탭 열기

claude.ai/download 에서 내려받아 설치하고, Pro 이상 계정으로 로그인해요.

앱 위쪽의 Code 탭을 클릭. 모델은 Opus 4.8 이상으로 맞추면 결과가 가장 좋습니다.

3

볼트 폴더 연결 — 바로 여기가 '연결'!

Code 탭에서 작업 환경을 Local(내 컴퓨터) 로 둡니다.

'Select folder'(폴더 선택)를 눌러, 1단계에서 만든 볼트 폴더를 그대로 선택해요.

이 순간 클로드와 옵시디언이 같은 폴더를 보게 됩니다. 연결 끝 — 따로 플러그인 설치도, 복사도 필요 없어요.

4

연결됐는지 확인

채팅창에 이렇게 물어보세요: "이 폴더 뭐야? 안에 뭐가 있어?"

클로드가 볼트 안 노트를 읽어 대답하면 연결 성공입니다. 이제 앞의 프롬프트들을 그대로 쓰면 돼요.

이게 전부예요. 위 4단계면 클로드가 볼트를 읽을 준비가 끝납니다. 특별한 플러그인도, 추가 설치도 필요 없어요 — 이제 앞의 프롬프트로 허브를 지으면 됩니다.
나부터 알기 · 경로 지도

야간비행의 실제 폴더 경로

볼트 최상위(C:\Deeper-era) 아래에 이런 폴더들이 있어요. '어디에 쌓이면 허브의 어느 방으로 흐르는지'를 한 표로 정리했습니다.

볼트 폴더 (창고)여기에 담기는 것흐르는 방향 → 허브의 방
00-Daily\매일 대화·데일리노트·모닝페이지·밤하늘 원본→ 데일리 / 밤하늘

02-Logic\ · 03-Rhetoric\논리 여정·발행한 글→ 여정 / 쓴 글
06-허브\펼쳐 보이는 HTML 페이지 전부= 무대 자체
06-허브\reading\북토크 안내·읽기 안내 1~3주 (책모임!)→ 함께읽기 (reading.html)

허브(06-허브) 안은 이렇게 생겼어요

index.html — 첫 화면. 북두칠성 지도(별 7개 = 방 7개)에서 원하는 방을 클릭.

seeds.html · nightsky.html · journey.html · reading.html · writings.html · oasis.html — 각각 방 한 개.

board\ — 자료실(안내문·가이드 모음). 책모임이라면 여기에 발제문·자료를 쌓으면 좋아요.

왜 HTML이냐면 — HTML은 인터넷 브라우저(크롬 등)가 바로 여는 '웹페이지 파일'이에요. 더블클릭만 하면 열리고, 나중에 인터넷에 올려 링크로 공유하기도 쉬워서 '펼쳐 보이는 무대'에 딱 맞아요.

후배에게 · 따라하기

책모임 리더를 위한 4단계

여러 관심사와 책모임 자료를 통합·정리·전시하고 싶은 후배에게 그대로 건네는 순서예요.

1

창고(볼트) 만들기 — 폴더부터

옵시디언을 설치하고( obsidian.md 에서 무료 ), 'Create new vault'로 폴더 하나를 정합니다. 예: C:\내서재.

그 안에 폴더 몇 개만: 매일기록 · 생각씨앗 · 책모임 · 허브. 처음엔 이 정도로 충분해요. 많이 만들 필요 없습니다.

2

자료 모으기 — 편하게 쌓기만

책모임 발제문, 밑줄 친 문장, 모임 후기, 관심사 메모를 각 폴더에 노트로 저장. 형식은 신경 쓰지 말고 내용부터.

연결하고 싶은 곳엔 [[책제목]] 처럼 링크를 걸어두면, 나중에 관계도가 저절로 생깁니다.

3

무대(허브) 짓기 — AI에게 부탁

여기가 핵심. HTML을 직접 코딩할 필요 없어요. 클로드 코드에게 아래 프롬프트를 복사해 부탁하면 페이지를 만들어 줍니다.

먼저 허브\index.html(첫 화면 지도) 하나, 그다음 책모임.html 방 하나 — 이렇게 하나씩 늘려가면 됩니다.

4

펼쳐 보이기 — 열고, 나누기

만들어진 index.html을 더블클릭하면 브라우저에서 바로 열려요. 이게 내 허브예요.

모임원과 나누고 싶으면 무료 배포 서비스(Netlify 등)에 올려 링크 하나로 공유. 이 단계도 AI에게 "이 폴더를 인터넷에 올려서 링크 만들어줘" 하고 부탁하면 됩니다.

공개 주의 — 개인 일기·모임원 실명·사적 메시지가 담긴 방은 배포 전에 꼭 빼거나 비공개로. 공개할 방만 골라 올리세요.

그대로 복사해 쓰는

프롬프트 모음

클로드 코드에게 그대로 붙여넣으면 됩니다. [대괄호] 안은 본인 상황으로 바꿔 쓰세요.

① 볼트 진단 — 내 자료부터 파악
언제: 이미 옵시디언에 자료가 좀 있을 때. AI가 구조를 먼저 이해하게 함.
내 옵시디언 볼트는 [C:\내서재] 폴더야.
안을 훑어보고 (1) 어떤 폴더·주제가 있는지 (2) 자료가 많은 곳과
비어 있는 곳 (3) 노트끼리 연결(링크)이 얼마나 되어 있는지를
초보자가 알기 쉽게 표로 정리해줘. 기존 파일은 절대 고치지 말고
읽기만 해.

연결했다면 경로는 생략 — 3단계에서 Select folder로 볼트를 연결했다면, 클로드가 이미 그 폴더를 보고 있어요. 첫 줄을 빼고 "이 볼트를 훑어보고…" 로 시작하면 됩니다. (경로를 다시 알려줄 필요 없음)

② 허브 첫 화면(지도) 만들기
언제: 무대의 표지이자 각 방으로 가는 입구를 만들 때.
내 볼트 자료를 한눈에 보여주는 허브 첫 화면을 HTML 한 장으로
만들어줘. 파일 위치는 [C:\내서재\허브\index.html].
- 맨 위에 제목 "[모임이름] 서재"와 한 줄 소개
- 아래에 방(페이지)으로 가는 카드 버튼: 책모임 · 생각씨앗 · 관심사
- 색은 [따뜻한 크림색 배경 + 짙은 초록 + 금색 포인트]로
- 코드는 몰라도 되니, 더블클릭하면 브라우저에서 바로 열리게
  한 파일 안에 다 담아줘(인터넷 없이도 열리게).
만들기 전에 어떤 방을 넣을지 먼저 나한테 물어봐 줘.
③ 책모임 방 만들기 — 자료 자동 정리
언제: 책모임 자료를 한 방에 모아 보여줄 때. (리더에게 가장 중요)
[C:\내서재\책모임] 폴더의 노트들을 읽어서, 책모임 방 페이지
[허브\책모임.html]를 만들어줘.
- 책별로 카드 하나씩: 책 제목 · 모임 날짜 · 발제 요약 · 밑줄 문장
- 최신 모임이 위로 오게 정렬
- 카드 클릭하면 그 책 상세로 펼쳐지게
- 폴더에 노트가 늘면 다시 실행만 해도 갱신되게 만들어줘
첫 화면(index.html)에서 이 방으로 넘어가는 링크도 연결해줘.
④ 관계도(그래프) 페이지 만들기
언제: 책·주제·사람이 어떻게 얽혀 있는지 그림으로 보이고 싶을 때.
내 볼트 노트들의 [[링크]] 연결을 읽어서, 어떤 주제·책·사람이
서로 이어져 있는지 보여주는 관계도 페이지를 HTML로 만들어줘.
- 점(노트) + 선(연결)으로 된 별자리 같은 그림
- 점에 마우스를 올리면 노트 제목이 뜨고, 클릭하면 그 방으로
- 연결이 많은 핵심 주제는 더 크고 밝게
초보자용으로 그림 아래에 "이 관계도 읽는 법"도 두세 줄 써줘.
⑤ 인터넷에 올려 링크로 공유
언제: 완성된 허브를 모임원과 나눌 때.
[C:\내서재\허브] 폴더를 인터넷에 올려서 링크 하나로 공유할 수
있게 해줘(무료 배포). 올리기 전에, 개인적이라 빼야 할 방이
있는지 먼저 목록으로 보여주고 나한테 확인받아줘.
다 되면 접속 주소를 알려줘.
먼저 확인받게 한 이유 — 배포는 되돌리기 어려운 '외부 공개'예요. 개인 자료가 섞이지 않았는지 사람이 한 번 눈으로 보고 넘어가는 안전장치입니다.

후배가 물어볼 만한 것들

Q. 코딩(HTML)을 배워야 하나요?
아니요. 위 프롬프트처럼 AI에게 말로 부탁하면 됩니다. 후배가 할 일은 자료를 옵시디언에 쌓는 것과 어떻게 보이면 좋을지 말로 설명하는 것뿐이에요.
Q. 옵시디언 없이도 되나요?
가능은 하지만 권하지 않아요. 옵시디언은 자료가 흩어지지 않게 한 폴더에 모아주고, [[링크]]로 관계도까지 만들어줘서 '통합 관리'라는 목적에 가장 잘 맞습니다. 무료이기도 하고요.
Q. 자료가 늘면 허브를 매번 다시 만들어야 하나요?
③번처럼 '폴더를 읽어 자동 정리'로 만들어두면, 노트를 추가한 뒤 AI에게 "다시 갱신해줘" 한마디면 됩니다. 무대가 창고를 다시 읽어 채워줘요.
Q. 어디서부터 시작하면 좋을까요?
가장 자료가 많은 것 하나(대개 책모임)부터. 방 한 개를 먼저 완성해 성취감을 느끼고, 그다음 관심사 방을 하나씩 늘리세요. 처음부터 방 7개를 다 만들려 하지 않는 게 요령입니다.
